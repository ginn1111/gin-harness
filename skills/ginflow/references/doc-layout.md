# Doc layout

Use target repo for project work.

Suggested layout:

```text
project/
├── AGENTS.md
├── briefs/
├── specs/
├── plans/
├── adrs/
└── src/
```

Guidelines:
- `briefs/` for task contracts
- `specs/` for behavior/contract detail
- `plans/` for execution order
- `adrs/` for decisions worth keeping
- skip folders you do not need
- prefer smallest artifact set that keeps handoff clear
